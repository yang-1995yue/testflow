# TestFlow 网络访问指南

## 1. 查看本地IP地址

### Windows系统
1. 按下 `Win + R` 组合键打开运行窗口
2. 输入 `cmd` 并按回车键打开命令提示符
3. 在命令提示符中输入 `ipconfig` 并按回车键
4. 查找 `无线局域网适配器 WLAN` 或 `以太网适配器 以太网` 部分
5. 找到 `IPv4 地址`，这就是您的本地IP地址（例如：192.168.1.100）

### macOS/Linux系统
1. 打开终端
2. 在终端中输入 `ifconfig`（macOS）或 `ip addr`（Linux）并按回车键
3. 查找活动网络接口（通常是 `en0` 或 `eth0`）
4. 找到 `inet` 地址，这就是您的本地IP地址（例如：192.168.1.100）

## 2. 局域网访问

经过我们的配置修改，TestFlow项目现在已经支持局域网访问：

### 前端访问
1. 确保您已经启动了TestFlow项目（运行 `run.bat` 或 `run.sh`）
2. 告诉您的同事在浏览器中输入：`http://您的本地IP地址:3000`
   例如：`http://192.168.1.100:3000`

### 后端API访问
如果需要直接访问后端API，同事可以使用：`http://您的本地IP地址:9000`
例如：`http://192.168.1.100:9000`

## 3. 公网访问解决方案

如果您需要让不在同一局域网的用户访问您的TestFlow项目，可以使用以下方法：

### 方法一：使用Ngrok（推荐）
Ngrok是一个简单易用的工具，可以将您的本地服务器暴露到公网。

#### 安装步骤：
1. 访问 [Ngrok官网](https://ngrok.com/) 注册账号并下载适合您系统的Ngrok
2. 解压下载的文件，将 `ngrok.exe`（Windows）或 `ngrok`（macOS/Linux）放在您的项目目录中
3. 打开命令提示符或终端，进入项目目录
4. 运行以下命令进行身份验证（需要您的Ngrok Auth Token，可在官网获取）：
   ```bash
   ngrok config add-authtoken <您的Auth Token>
   ```

#### 启动步骤：
1. 确保TestFlow项目已经在本地正常运行
2. 打开命令提示符或终端，进入项目目录
3. 运行以下命令暴露前端服务：
   ```bash
   ngrok http 3000
   ```
4. 运行以下命令暴露后端服务：
   ```bash
   ngrok http 9000
   ```
5. Ngrok会为您生成公网访问地址（例如：`https://abcdef1234.ngrok.io`）
6. 将这些地址分享给需要访问的用户

### 方法二：使用花生壳
花生壳是国内常用的动态域名解析工具，支持将本地服务器映射到公网。

#### 使用步骤：
1. 访问 [花生壳官网](https://hsk.oray.com/) 注册账号并下载客户端
2. 安装并登录花生壳客户端
3. 添加映射，设置以下参数：
   - 映射名称：TestFlow前端
   - 应用类型：HTTP
   - 内网主机：您的本地IP地址
   - 内网端口：3000
4. 重复上述步骤，添加后端服务的映射（内网端口改为9000）
5. 花生壳会为您生成公网访问地址

## 4. 注意事项

1. 确保您的防火墙设置允许外部访问3000和9000端口
2. 局域网访问时，所有设备需要连接到同一个路由器
3. 公网访问时，请注意网络安全，避免暴露敏感数据
4. 如果使用Ngrok等工具，免费版的访问地址会定期变化
5. 建议在公网访问时使用HTTPS协议（Ngrok提供免费HTTPS）

## 5. 故障排除

### 问题：同事无法访问您的项目
1. 检查您的IP地址是否正确
2. 确保TestFlow项目正在运行
3. 检查防火墙设置，确保3000和9000端口已开放
4. 尝试关闭防火墙后再次测试

### 问题：公网访问速度慢
1. 公网访问速度受您的网络带宽限制
2. 免费版的Ngrok等工具可能有速度限制
3. 尝试使用付费版的公网穿透工具

如果您遇到其他问题，请参考相关工具的官方文档或寻求技术支持。
